  Streaming KV cache: first call does full prompt eval (~10 s for a 5-token prompt), then each
  generated token is one attention step over the cached K/V (predict_step_kv_bytes).
  KV state round-trips through List[Bytes] between steps so the WASM bump allocator
  stays flat — JS copies the caches out, __heap_restores the scratch, and feeds them
  back in. Sampling (temperature + top-k) happens inside Almide: JS supplies a PRNG-derived
  rand01 each step. Greedy = temp 0 or top-k 1.
  Each weight matrix stays packed in Q1_0 throughout via
  matrix.linear_q1_0_row_no_bias. Model is cached in IndexedDB on first run.
